K-12 Online Arduino Block Code

Overview

KOABC will be a web application that allows students to write arduino code using a block coding interface. There will be multiple code complexities, optional lessons, and arduino file downloading / uploading.

The Problem

Computer science and engineering don't have as much outreach to K-12 students as with other fields. In addition, the outreach that is being done in these areas can sometimes scare students off by being too complex, or bore students by being too easy. The core question to this conundrum is as follows: how can we make an applicaiton that performs effective student outreach for the BOREALIS lab?

Success

Students learn about new fields and get hands on experience in Computer Science and Engineering. Student can use a tool to create arduino block code that is tailored specifically to the circuit boards developed by the BOREALIS lab. This prevents an overwhelming number of options for students, while still allowing learning and self expression.

Approach

To allow the greatest accessibility to the application, a website will be used. The website will need to be reactive, so the REACT web framework will be used as a basis. For deploying the website, AWS will be used to decrease the amount of upkeep needed.

Discovery

User Surveys

To start with, surveys were performed with both BOREALIS team members. These surveys acted as a baseline for the project goals and features.

Brainstorming

Ideas were written on physical sticky notes for the project approach. Once all of the ideas were written out, they were placed into an excel document and given scores for feasibility, importance, and expense. This allowed the more important ideas to float to the surface, while still allowing all ideas to be represented.

Prototyping

Low-Fidelity

To start with, sketch prototypes were created of possible drag and drop interfaces with code blocks that look like puzzle peices. This would begin as a generic idea to start from with implementation that could be expanded on with new ideas for less complex implementations.

High-Fidelity

Since this project is heavily reliant on interactive design, most high-fidelity prototyping tools would not be useful for it. Instead, development has been started in React to test different systems and UI design approaches.

>

Testing

Usability Tests

Testing will be done over the period of a month in the spring with highschoolers in the Bozeman area. These will be performed using the React development version of the website.

Conclusion

This project is still ongoing, though it will be completed by the end of March 2025.
